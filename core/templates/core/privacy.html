Privacy Policy

Privacy controls for sensitive documents.

This template explains how the DocSeva deployment is intended to handle user information. Replace this page with reviewed legal text before public commercial launch.

Information stored

DocSeva stores account details, profile details, uploaded document metadata, uploaded files, renewal information, support tickets, notifications, security logs, and optional emergency QR information.

Document sharing

Documents are private by default. Public document access requires an enabled share link. Users can disable sharing and can decide whether public download is allowed.

Emergency QR sharing

Emergency QR pages expose only the emergency fields the user enters and only when emergency sharing is enabled.

Security logs

The application records important actions such as login attempts, uploads, downloads, sharing changes, and admin actions. Logs are used for account safety and audit review.

Production responsibility

The deployer must configure secure hosting, strong environment variables, HTTPS, a protected database, reliable backups, and real contact information before serving real users.
